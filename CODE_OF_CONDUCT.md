# Code of Conduct

The squid4win project aims to be practical, respectful, and technically useful
for maintainers, contributors, and users.

## Expected behavior

- Be respectful and constructive in issues, pull requests, reviews, and release
  discussions.
- Focus feedback on the work, the evidence, and the repository contracts.
- Assume good faith while still being precise about bugs, regressions, and
  validation gaps.
- Cite files, logs, workflow runs, or reproduction steps when you disagree with
  a change.
- Respect licensing, attribution, and project boundaries.

## Unacceptable behavior

- Harassment, insults, threats, or discriminatory language.
- Repeated bad-faith disruption of reviews, issue threads, or release
  discussions.
- Publishing private information, secrets, credentials, certificates, or
  security-sensitive material that should not be public.
- Knowingly misrepresenting validation results, release status, or the current
  repository state.
- Spam, impersonation, or deliberate attempts to derail collaboration.

## Scope

This code of conduct applies to public project spaces for squid4win, including
issues, pull requests, reviews, discussions, and workflow-related collaboration
around this repository.

## Reporting

If GitHub moderation or private maintainer contact features are available, use
them first for conduct concerns.

If a public issue is the only practical option, keep it minimal, avoid sensitive
details, and ask maintainers for a private follow-up.

## Enforcement

Maintainers may edit, hide, lock, or remove content, and may restrict
participation, when behavior harms contributors or the project.
